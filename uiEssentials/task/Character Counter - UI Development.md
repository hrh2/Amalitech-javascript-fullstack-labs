# UI Essentials: Character Counter
## Learning Objectives

* **HTML & CSS Mastery:** Build a pixel-perfect user interface using semantic HTML and modern CSS.
* **Responsive Design:** Create a layout that adapts seamlessly to different screen sizes.
* **UI/UX Best Practices:** Implement a visually appealing and user-friendly design that matches the provided mockups.

## Introduction

In this lab session, you will build the user interface for the Character Counter web app. Your goal is to replicate the provided design as closely as possible. The focus will be on creating a well-structured HTML layout and styling it with CSS to ensure responsiveness and visual fidelity.

## Project Setup

1. **Get the project files:** Download the starter files provided by your trainer.
2. **Review the assets:** Familiarize yourself with the provided HTML, CSS, and design mockups to understand the desired layout and style.

## Tasks

### 1. HTML Structure

* Use semantic HTML elements to create a clear, accessible layout.
* Include an input field (textarea or text input) for user text entry.
* Create an area to display the character count.

### 2. CSS Styling

* Style the UI to match the provided design closely.
* Ensure that the layout is responsive across various devices.
* Optionally, use a CSS preprocessor (e.g., Sass or Less) for better organization.

### 3. Testing and Refinement

* Verify that the design matches the mockup pixel-by-pixel.
* Test the layout on different screen sizes to ensure responsiveness.
* Refine the styling based on your own review and any feedback from peers.
